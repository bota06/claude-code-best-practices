## なぜこの順番か

```
理由1：Hooksは確定的に動く（100%実行）
       CLAUDE.mdはアドバイザリー（約80%遵守）
       → CCが誤操作しても自動ブロックできる仕組みを先に作る

理由2：CLAUDE.mdがないとCCは毎回ゼロから文脈を理解する
       → 早期に整備するほどセッション品質が上がる

理由3：Specなしの実装はCCが合理的な仮定を立てるが間違いが多い
       → 実装前に仕様を固めると手戻りが激減する
```
