# ドキュメント管理ルール

## 原則: 導出可能な情報は書かない

以下はドキュメントに記録しない。必要なときにツールで取得する。

| 情報 | 取得方法 |
|---|---|
| Git状態（ブランチ・コミット・差分） | `git status`, `git log` |
| 実装済み内容・変更履歴 | `git log` |
| ファイル一覧・ファイル状態 | `ls`, `glob` |
| テスト件数 | テストコマンド実行 |
| コードの構造・依存関係 | コードを読む |

## 情報の管理先

| 種類 | 管理先 |
|---|---|
| ルール・ワークフロー・指示 | `CLAUDE.md`（簡潔に） |
| 特定テーマのルール | `.claude/rules/*.md` |
| 設計判断の "Why"・経緯 | Auto Memory（`~/.claude/projects/*/memory/`） |
| 現在のタスク・次にやること | `.claude/rules/current-task.md` |
| ドメイン知識（ペルソナ等） | `knowledge/*.txt`（一元管理） |

## やってはいけないこと

- CONTEXT.md のような「全部入り状態ファイル」を作らない
- 同じ情報を複数箇所に書かない（Single Source of Truth）
- git log で追える実装記録をドキュメントに転記しない
- ファイル一覧やスキーマをドキュメントで管理しない（実ファイルが正）
